# CLAUDE.md

Guidance for agents working in OpenUSDZEditor ŌĆö a native macOS USDZ viewer/editor (SwiftUI + RealityKit + embedded Python/`usd-core`).

## Concurrency (read first)

Multiple agents work this repo at the same time. **Never run `git stash`** ŌĆö it mutates shared working-tree state and clobbers other agents' uncommitted changes. Isolate your work on a topic branch (or a worktree) instead. Never commit to `main` directly; land changes via PR with green CI.

## Testing ŌĆö 100% coverage

New behavior ships with tests, and logic modules hold a **100% line-coverage floor** (95% for a few). Coverage is CI-enforced per module via `scripts/coverage-gate.sh`; do not lower a floor or add override labels. MeshKit additionally runs a fuzz corpus (`FuzzCorpus.swift`) ŌĆö extend it for fuzz-sensitive changes. See `specs/testing.md` for per-module policy.

Run the full local check set (mirrors CI) before opening a PR:

```sh
bash scripts/dependency-lint.sh       # one-directional dependency rules
bash scripts/module-governance.sh     # module governance gate
bash scripts/fetch-python-runtime.sh  # one-time: usd-core runtime
bash scripts/test-all.sh --coverage   # all package tests
bash scripts/coverage-gate.sh         # coverage floors + fuzz corpus
```

## Architecture ŌĆö modern Swift (2026)

Swift 6.0, strict concurrency, macOS 14+. Follow current best practices: value types and `actor`/`Sendable` isolation over shared mutable state, `async/await` (no callback pyramids or legacy GCD), dependency injection over singletons, protocol-oriented extension points, and `@Observable`/`@MainActor` SwiftUI. Keep new code `Sendable`-clean rather than silencing data-race diagnostics.

The workspace is modular SPM packages under `Packages/` with **strictly one-directional dependencies** enforced by `scripts/dependency-lint.sh`. Layering (a module may only import what its policy allows):

- `USDCore`, `MeshKit`, `DicyaninDesignSystem` ŌĆö leaf modules, zero internal deps. `USDCore`/`MeshKit` stay pure Swift (no UI/GPU/Python).
- `USDBridge`, `ConversionKit`, `ValidationKit`, `ScriptingKit` ŌåÆ `USDCore`.
- `ViewportKit` ŌåÆ USDCore, MeshKit. `EditingKit` ŌåÆ USDCore, ValidationKit, MeshKit.
- `AgentMCP` ŌåÆ the kits (never `EditorUI`). `EditorUI` ŌåÆ all kits + DesignSystem. `App`/`CLI` ŌåÆ top of the graph.

Adding a package requires a matching policy entry in `dependency-lint.sh` **and** an update to `specs/architecture.md` ŌĆö an ungoverned module is itself a lint failure. Specs in `specs/` are the contract; keep them in sync with code.

## Layout

- `App/` ŌĆö macOS app shell. The `.xcodeproj` is generated by XcodeGen from `project.yml` (git-ignored); edit `project.yml`, then `bash scripts/generate-xcodeproj.sh`. SPM packages remain the source of truth for library code.
- `CLI/` ŌĆö the `openusdz` command-line tool (same pipeline as the app; also serves the MCP server).
- `Packages/` ŌĆö Swift library packages.
- `Resources/Python/` ŌĆö bundled `usd-core` bridge scripts.
- `specs/`, `PRD.md`, `ROADMAP.md` ŌĆö design contract and phase status.

## PRs

Topic branch ŌåÆ PR against `main`, squash-merged. Use a conventional-commit-style title (e.g. `feat(viewport): ŌĆ”`, `fix(meshkit): ŌĆ”`). Both `lint` and `test` CI jobs must pass.
